# Coding Rules

## General Rules

- strict TypeScript only
- no any
- reusable architecture
- maintainability first

---

## Component Rules

- no API calls inside UI components
- no business logic in UI
- small focused components
- reusable structure only

---

## Naming Rules

Components:
PascalCase

Hooks:
useSomething

Services:
something.service.ts

---

## Performance Rules

- lazy loading where appropriate
- optimized rendering
- minimal re-renders
- responsive images

---

## Quality Standard

Code must feel production-ready.
